Remarks

When the chained monad is executed, if an error is returned from the first monad, then the second won't get run. The design is similar to Maybe Monad in certain functional language, which provides a less verbose way of error handling.

Examples
            
var m1 = new SimpleMonad<string>("12345");
var m2 = m1.Then( v => new ParseStringMonad(v));
yield return m2.Do();
